# IHDP1000 数据集使用模式对比分析

## 概述

IHDP1000 数据集包含 1000 个独立的 replications，每个 replication 有 672 个训练样本和 75 个测试样本。本文档对比两种使用模式的差异。

## 3. 详细对比

### 3.1 数据处理差异

| 特性 | 合并模式 | 分离模式 |
|------|---------|---------|
| **训练样本数** | 604,800 | 470 × N (N=replications数) |
| **测试样本数** | 750,000 | 75 × N |
| **模型数量** | 1 | N (每个replication一个) |
| **训练时间** | 长（单次大数据集训练） | 更长（N次独立训练） |
| **内存占用** | 高（一次性加载所有数据） | 低（每次只加载一个replication） |

### 3.2 模型行为差异

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        合并模式: 单一平均模型                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  输入特征: [x1, x2, ..., x25]                                           │
│         │                                                                │
│         ▼                                                                │
│  ┌────────────────────────────────────────┐                             │
│  │         单一 CEVAE 模型                │                             │
│  │  学到的参数: θ_avg (所有数据平均)       │                             │
│  └────────────────────────────────────────┘                             │
│         │                                                                │
│         ▼                                                                │
│  输出预测: ITE ≈ 4.4 (对所有样本)                                        │
│                                                                          │
│  优点: 训练一次，预测快速                                                 │
│  缺点: 无法捕捉个体差异，ITE 误差巨大                                     │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│                        分离模式: N个独立模型                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  Rep 1 数据 → Model 1 → ITE ≈ 3.5 (适合 Rep 1)                          │
│  Rep 2 数据 → Model 2 → ITE ≈ 4.2 (适合 Rep 2)                          │
│  ...                                                                      │
│  Rep N 数据 → Model N → ITE ≈ 2.8 (适合 Rep N)                          │
│                                                                          │
│  优点: 每个模型适应自己的数据分布，个体预测准确                            │
│  缺点: 需要训练 N 个模型，时间和计算成本高                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.3 评估结果对比

| 指标 | 合并模式 | 分离模式 | IHDP基准 |
|------|---------|---------|----------|
| **训练 ITE** | 14.530 | 1.952 ± 0.635 | 1.980 ± 0.660 |
| **训练 ATE** | 0.009 | 0.240 ± 0.063 | 0.475 ± 0.211 |
| **训练 PEHE** | 9.945 | 2.472 ± 1.325 | 2.473 ± 1.351 |
| **测试 ITE** | 14.697 | 1.813 ± 0.435 | 1.531 ± 0.355 |
| **测试 ATE** | 0.018 | 0.407 ± 0.124 | 0.340 ± 0.110 |
| **测试 PEHE** | 10.065 | 2.082 ± 0.882 | 2.023 ± 0.969 |

**关键观察:**
- 分离模式的 ITE 和 PEHE 与 IHDP 基准一致
- 合并模式的 ITE 和 PEHE 是分离模式的 5-7 倍
- 分离模式提供了标准差，可以评估模型稳定性

### 3.4 为什么合并模式 ATE 更小？

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        ATE 计算差异                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  合并模式:                                                               │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │ 真实情况:                                                        │    │
│  │   200 个样本来自 ATE=2 的 replications, 真实效应 ≈ 400            │    │
│  │   200 个样本来自 ATE=4 的 replications, 真实效应 ≈ 800            │    │
│  │   200 个样本来自 ATE=6 的 replications, 真实效应 ≈ 1200           │    │
│  │   总真实效应 = 2400, 平均 ATE = 4.0                               │    │
│  │                                                                  │    │
│  │ 模型预测: 所有人 ATE ≈ 4.0 (平均), 预测效应 = 2400                │    │
│  │                                                                  │    │
│  │ ATE 误差 = │预测 - 真实│ = │2400 - 2400│ / 600 = 0.0             │    │
│  │           (平均正确!)                                               │    │
│  │                                                                  │    │
│  │ ITE 误差 = │预测 - 真实│ / 600                                  │    │
│  │          = │(2400/600) - 各自真实值│ 的平均                       │    │
│  │          = │4.0 - 2.0│ + │4.0 - 4.0│ + │4.0 - 6.0│ 的平均         │    │
│  │          ≈ 2.0 + 0.0 + 2.0 = 4.0 (巨大!)                         │    │
│  └─────────────────────────────────────────────────────────────────┘    │
│                                                                          │
│  分离模式:                                                               │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │ 每个 replication 独立:                                           │    │
│  │   Rep 1: 真实 ATE=2.0, 模型学到 1.9, ITE误差 ≈ 0.5              │    │
│  │   Rep 2: 真实 ATE=4.0, 模型学到 4.1, ITE误差 ≈ 0.5              │    │
│  │   Rep 3: 真实 ATE=6.0, 模型学到 5.9, ITE误差 ≈ 0.5              │    │
│  │                                                                  │    │
│  │   平均 ATE 误差 ≈ │1.9-2.0│ + │4.1-4.0│ + │5.9-6.0│ ≈ 0.1        │    │
│  │   平均 ITE 误差 ≈ 0.5 (小!)                                      │    │
│  └─────────────────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────────────────┘
```

## 4. 使用建议

### 4.1 何时使用合并模式？

```
✓ 适用场景:
  - 快速原型开发
  - 测试模型在大规模数据上的可扩展性
  - 只关心平均处理效应 (ATE)
  - 计算资源有限

✗ 不适用场景:
  - 需要准确的个体因果效应预测
  - 需要评估模型的异质性处理效应能力
  - 需要与其他研究进行公平比较
```

### 4.2 何时使用分离模式？

```
✓ 适用场景:
  - 正式的研究和论文发表
  - 需要准确的个体因果效应 (ITE)
  - 需要评估模型稳定性 (标准差)
  - 需要与 IHDP 等基准数据集比较
  - 研究不同 replications 间的差异

✗ 不适用场景:
  - 快速迭代开发
  - 计算资源非常有限
```

### 4.3 推荐实践

```bash
# 研究和论文: 使用分离模式，10-100 replications
python cevae_ihdp.py -dataset ihdp1000 -separate_reps -n_reps 100

# 快速测试: 使用合并模式
python cevae_ihdp.py -dataset ihdp1000

# 与 IHDP 比较: 使用相同的 replications 数
python cevae_ihdp.py -dataset ihdp1000 -separate_reps -n_reps 10
python cevae_ihdp.py -dataset ihdp -reps 10
```

## 5. 结论

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         核心结论                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │

│     - 是正确的 IHDP1000 使用方式                                         │
```

## 附录：数据结构细节

### A. IHDP1000 原始数据结构

```python
# 训练集
x_train:  (672, 25, 1000)  # 特征是 3D 的
t_train:  (672, 1000)      # 处理是 2D 的
yf_train: (672, 1000)      # 结果是 2D 的
mu0_train: (672, 1000)     # 潜在结果0是 2D 的
mu1_train: (672, 1000)     # 潜在结果1是 2D 的

# 测试集
x_test:   (75, 25, 1000)
t_test:   (75, 1000)
yf_test:  (75, 1000)
mu0_test: (75, 1000)
mu1_test: (75, 1000)
```

### B. 合并模式转换

```python
# 特征: 3D → 2D
x_merged = x_train[:, :, :n_reps].transpose(0, 2, 1).reshape(-1, 25)
# (672, 25, 1000) → (672, 1000, 25) → (672000, 25)

# 其他: 2D → 2D (转置)
t_merged = t_train[:, :n_reps].T.reshape(-1, 1)
# (672, 1000) → (1000, 672) → (672000, 1)
```

### C. 分离模式提取

```python
# 每个 replication
for i in range(n_reps):
    x_rep = x_train[:, :, i]      # (672, 25)
    t_rep = t_train[:, i]         # (672,)
    y_rep = yf_train[:, i]        # (672,)
    # ... 训练独立模型
```
